# Cursor Rule Library

Эта папка хранит переносимую библиотеку правил Cursor, не привязанную к конкретному проекту.

## Как это устроено
- `/cursor-rule-library/shared` — canonical source of truth для переносимых правил.
- `/.cursor/rules` — активные правила проекта, которые реально читает Cursor.

## Почему active rules лежат плоско в `/.cursor/rules`
Активные `.mdc` files нужно хранить без вложенных подпапок внутри `/.cursor/rules`, чтобы избежать проблем с применением правил.

## Как переносить в другой проект
1. Скопировать нужные `.mdc` файлы из `/cursor-rule-library/shared`.
2. Вставить их в `/.cursor/rules` нового проекта.
3. Убедиться, что в каждом правиле стоит `alwaysApply: true`.

## Зачем это нужно
Такой подход позволяет:
- хранить переносимый набор правил отдельно от проекта;
- быстро переносить его между репозиториями;
- иметь активные project rules без ручного переписывания текста.
