# Frontend UI Direction

TimChat frontend should follow a Cursor-inspired design language:
light-theme-first, minimalistic, dense, professional, desktop-first, and editor-like.

The goal is not to copy Cursor literally, but to achieve a similar feeling of a high-quality operational workspace:
compact controls, clean typography, subtle borders, strong layout hierarchy, contextual side panels, and low visual noise.

The interface must prioritize operational usability for realtime collaboration workflows:
workspace navigation, channel hierarchy, chat timeline, member presence, call participation, moderation controls, permission-aware actions, audit/context panels, and reconnect/session states.

This product should feel like a serious daily work tool for communication and coordination, not like a decorative dashboard or a marketing-style messenger.
