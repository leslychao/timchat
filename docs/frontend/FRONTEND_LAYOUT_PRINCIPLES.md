# Frontend Layout Principles

TimChat UI should be structured as a dense desktop workspace with clear operational zones:

- left navigation rail for workspace switching
- secondary sidebar for channels / sections / threads where applicable
- main central area for chat timeline or active room content
- right contextual panel for members, presence, moderation, audit, pinned items, or call details
- compact top bar for current context, search, and quick actions

The UI should avoid oversized cards, excessive whitespace, decorative gradients, and mobile-first dashboard composition.

Priority should be given to:
- fast scanning
- high information density
- predictable interaction zones
- keyboard-friendly workflows
- low-noise state presentation
- clear distinction between stable context and live-changing context
